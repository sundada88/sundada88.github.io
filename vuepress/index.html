# 使用 vuepress + Github page 搭建技术博客

# 1. 本地搭建

  1. 创建一个本地目录
  mkdir blog && cd blog
  1. 使用包管理工具npm或者yarn在本地安装vuepress
  npm init -y 或者 yarn init -y
  npm install vuepress -D 或者 yarn add vuepress -D
  1. 创建首页文档
  mkdir docs && echo '# Hello VuePress' > docs/README.md
  1. 在package.json中添加一下运行指令
{
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}
  1. 执行docs:dev命令开启本地8080端口的服务器
npm run docs:dev 或者 yarn docs:dev

# 2. 基础配置

  1. 在 docs下面新建.vuepress文件夹，所有vuepress的配置信息都在这里,然后在.vuepress文件夹下新建config.js, 此时的项目结构是

├─ docs │ ├─ README.md │ └─ .vuepress │ └─ config.js └─ package.json├─ docs │ ├─ README.md │ └─ .vuepress │ └─ config.js └─ package.json

  1. 给config.js文件中添加内容
module.exports = {
  title: '能能洋的博客',
  description: '博客 blog vuepress'
}
  1. 在右上角添加导航栏
module.exports = {
  title: '能能洋的博客',
  description: 'Just playing around',
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      {
        text: '能能洋的博客',
        items: [{ text: 'Github', link: 'https://github.com/sundada88' }]
      }
    ]
  }
}
  1. 在docs下面新建README.md文件，添加如下内容
---
home: true
heroImage: /logo.gif
heroText: Blog 标题
tagline: Blog 副标题
actionText: 快速上手 →
actionLink: /zh/guide/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。
- title: Vue驱动
  details: 享受 Vue + webpack 的开发体验，在 Markdown 中使用 Vue 组件，同时可以使用 Vue 来开发自定义主题。
- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。
footer: MIT Licensed | Copyright © 2018-present Evan You
---

# 3. 在导航栏添加内容

  1. 比如在导航栏添加分享页面，在vuepress.config.js中添加如下内容
  themeConfig: {
    nav: [
      // ...
      { text: '分享', link: '/share/' },
      // ...
    ]
  }
  1. 点击分享按钮的时候，我们希望调到响应的页面，那我们需要在vuepress.config.js，添加 link对应的跳转页面
  themeConfig: {
    // ...
    sidebar: {
      '/share/': ['', 'share1', 'share2']
    }
    // ...

# 4. 配合 github page 生成博客网站

  1. 在根目录下创建deploy.sh, 内容如下：
#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy11'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master // 将 USERNAME 换成你的 github 的 name

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -
  1. 在 github 新建 repository, 名称叫做你的 <USERNAME.github.io>
  2. 运行 npm run build, 回生成一个 dist文件夹
  3. 运行 bash ./deploy.sh 将生成的 dist 内容推到 repository
  4. 你就可以在 https://<UNSRENAME>.gihub.io 看到你的博客内容了
